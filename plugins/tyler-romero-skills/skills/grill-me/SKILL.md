# Grill Me

Inspect the available code, documents, and artifacts before asking questions. Resolve anything discoverable from those sources directly instead of asking the user.

Build a decision tree for the plan, including dependencies between choices. Then:

1. Ask exactly one focused question at a time.
2. State the recommended answer and its key tradeoff before asking.
3. Incorporate the user's response into the shared understanding.
4. Follow each material branch until its dependencies and consequences are resolved.
5. Challenge vague goals, missing constraints, untested assumptions, and unclear success criteria.

Stop when no material ambiguity remains. Summarize the agreed decisions, remaining risks, and any deliberately deferred choices.
